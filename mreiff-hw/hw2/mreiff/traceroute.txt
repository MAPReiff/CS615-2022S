Comments/Annotations:
- When attempting to connect to www.stevens.edu, the IPv4 address that is
  being connected to is 104.18.130.28. This can be verified as both instances
  are attempting to reach this address, as well as using the dig command to
  veiry the DNS A records are pointing to this address.

- In addition to the IP address for the Stevens website, both instances also
  connect with multiple IPv4 address that start with `216.182`. These addresses
  are allocated to Amazon. As both instances pass through multiple of these
  addresses, it can be assumed that these are the addresses of different
  switches and routers that must be passed through in order for data to
  both leave the internal network as well as come into the internal network.

- Both instances also make connections with addresses that start with both
  `240.` and `241.`. When looking up the owner of these addresses, the results
  declare it to be a private IP address. According to Wikipedia, 240.0.0.0/4
  (240.0.0.0–255.255.255.254) is "[r]]eserved for future use", however I am
  assuming Amazon is using these addresses to connect to servers internally,
  as these addresses are not accessable directly from the internet.
  https://en.wikipedia.org/wiki/Reserved_IP_addresses
